# Google Business Profile / LINE Official Account 官方規格研究整理

日期：2026-03-29

目的：
- 仔細確認 Google 官方對商家文案與圖片 / 影片的要求
- 詳細整理 LINE 官方帳號常用文案欄位與圖片標準尺寸
- 將可直接執行的規格記錄下來，供後續 SOP、素材收集與製作使用

---

## 一、結論先看

### Google Business Profile

Google 商家最核心的文案 / 欄位，不是先做廣告圖，而是先把真實世界資訊補齊：

- 商家名稱
- 類別
- 地址或服務區域
- 電話
- 網站
- 營業時間
- 商家描述
- 服務項目
- 照片 / 影片
- 貼文（可選，但強烈建議）

Google 對內容最在意的是：

- 名稱要對應真實世界招牌與品牌
- 地址要真實且可驗證
- 描述要以介紹商家為主，不要塞連結或促銷價
- 圖片要真實、清楚、不要過度修圖或 AI 感太重

### LINE Official Account

LINE 官方帳號比較像是「品牌承接頁 + 對話轉換工具」，要準備的內容分成兩類：

1. 文案與欄位
   - 帳號名稱
   - 狀態文字
   - Profile 頁基本資料
   - 按鈕文案
   - 歡迎訊息
   - 自動回覆訊息
   - Rich menu 導覽文案
   - Rich message / 卡片訊息文案

2. 圖片與影音素材
   - 頭像
   - Profile Page 封面
   - Cover 圖
   - Rich message 圖
   - Rich video 素材
   - Card message 圖
   - Rich menu 圖
   - LINE VOOM 圖 / 影片 / 預覽圖

## 二、Google Business Profile 官方要求

### A. Google 建立 / 維護商家時最基本要補的資料

依 Google Business Profile 官方說明，商家建立與後續維護最關鍵的資訊包括：

- 商家名稱
- 地址、服務區域或地圖位置
- 電話
- 網站
- 營業時間
- 類別
- 照片與影片

Google 官方也明確說明：只有和客戶有面對面接觸的店家 / 服務型商家，才適合建立 Business Profile；純線上商家不適用。

### B. Google 文案欄位與內容規範

#### 1. 商家名稱

官方要求：

- 名稱必須反映商家在真實世界中使用的名稱
- 必須與招牌、網站、名片、文具、品牌識別一致
- 不可在名稱中塞入：
  - 行銷標語
  - 營業時間
  - 電話號碼
  - 網址
  - 無關鍵字
  - 不必要的特殊符號

這代表：

- 不要把「課程名 + 地區 + 電話 + 招生中」直接當成 Google 店名
- 若實際現場是公司名，Google 主名稱應優先以公司名或實體招牌名為準

#### 2. 商家描述（Business description）

官方要求：

- 用來介紹你提供什麼服務 / 產品、你的特色、使命、背景或歷史
- 字數上限：`750` 字元
- 不可包含：
  - URL / HTML
  - 促銷內容
  - 價格、折扣、特賣導向內容
  - 低品質、無關、誤導內容

因此 Google 描述應寫：

- 你是誰
- 你提供什麼
- 適合誰
- 你的專長 / 特色

不應寫成：

- 招生中、限時折扣、現在報名只要多少錢

#### 3. 類別（Category）

官方要求：

- 選最符合核心業務的主類別
- 所有同類型據點應維持一致
- 主類別之外可再加最多 `9` 個額外類別
- 不要把類別當關鍵字欄位亂塞

#### 4. 地址 / 服務區域

官方要求：

- 地址必須是完整、精準、真實存在的地址
- 不可用虛擬辦公室冒充實體據點
- 若是不在店址接待客人的服務型商家，應隱藏地址並使用服務區域
- 若顯示地址，該地址應有固定商家招牌

#### 5. 網站與電話

官方要求：

- 電話應連到該商家實際地點或服務據點
- 網站應對應該商家實際據點或品牌

#### 6. 服務項目（Services）

Google 官方允許填寫：

- 服務名稱
- 價格
- 描述

但自訂服務名稱若包含以下內容，可能被系統拒絕：

- 粗俗字詞
- 亂碼
- 個資
- 價格
- 電話號碼

所以建議：

- 服務名稱保持乾淨
- 價格寫在價格欄位 / 服務細節欄位，而不是硬塞進服務名稱

#### 7. 貼文（Posts）

Google 官方可建立 3 類常見貼文：

- Update
- Offer
- Event

官方規範重點：

- 貼文可含文字、照片或影片
- Offer 必填：標題、日期與時間
- Event 必填：標題、日期與時間
- Update 可放照片 / 影片 / 描述 / CTA 按鈕
- 貼文描述內若有電話號碼，可能被拒絕
- 超過 `6` 個月的貼文通常會被封存（若未設日期區間）

### C. Google 圖片 / 影片標準規格

#### 1. 照片

Google 官方規格：

- 格式：`JPG`、`PNG`
- 檔案大小：`10 KB` 到 `5 MB`
- 建議解析度：`720 x 720 px`
- 最低解析度：`250 x 250 px`
- 品質要求：
  - 對焦清楚
  - 光線充足
  - 不可過度修圖
  - 不可過度濾鏡
  - 要真實反映現場

#### 2. 影片

Google 官方規格：

- 長度：最長 `30` 秒
- 檔案大小：最大 `75 MB`
- 解析度：`720p` 以上

#### 3. Google Logo / Cover / Business photos

Google 目前在商家端支援：

- Logo
- Cover photo
- Business photos

但 Google 官方也提醒：

- 你設定的 cover photo 不保證一定會成為第一張主圖
- 若 cover 品質不好或其他來源照片更代表商家，Google 可能顯示其他照片

#### 4. Google 建議最低照片量

Google 官方對商家照片類型有最低建議量：

- 外觀照：至少 `3` 張
- 內部照：至少 `3` 張
- 產品照：至少 `3` 張
- 工作情境照：至少 `3` 張
- 團隊照：至少 `3` 張

對於課程 / 培訓型商家，可轉譯為：

- 外觀 / 招牌 / 建物入口：至少 3 張
- 教室 / 上課空間：至少 3 張
- 課程與實作情境：至少 3 張
- 工具 / 設備 / 教材：至少 3 張
- 講師 / 團隊：至少 3 張

## 三、LINE Official Account 官方要求

### A. LINE OA 需要準備的主要文案欄位

依 LINE 官方頁面與說明文件，店家經營 LINE OA 時，至少應準備以下文案與內容類型：

#### 1. 帳號基本資料 / Profile Page 文案

官方可設定的內容包括：

- 狀態（Status）
- 按鈕（Buttons）
- 營業時間（Business hours）
- 聯絡方式（Contact）
- 基本資料（Basic information）

此外 Profile Page 還能加入內容型 plug-ins，例如：

- 自由文字
- 內容列表
- 基本資料
- 優惠券
- 集點卡
- Delivery
- 最新貼文

官方限制：

- 最多 `20` 個 plug-ins
- `內容列表` 可放最多 `100` 項

#### 2. 狀態文字（Status）

LINE 官方教材明確寫到：

- Status 不應超過 `20` 個字元

這是目前我查到最明確、最實用的 LINE OA 文案字數硬限制之一。

#### 3. 歡迎訊息（Greeting message）

LINE 官方說明：

- 新好友加入後，Greeting Message 會自動發送
- 可由管理者啟用或停用
- 可插入顧客顯示名稱
- 可使用最多 `5` 個 balloons
- 可放入：
  - 文字
  - 貼圖
  - 圖片
  - 優惠券
  - Rich message
  - Rich video
  - 語音訊息

這代表 Greeting Message 不是只有一段文字，而是可以是一個小型 onboarding 序列。

#### 4. Auto-response messages

LINE 官方產品頁說明：

- Auto-response messages 用於管理員不在線、無法即時回覆、或非營業時段
- 可代替人工回覆，用於基本詢問與導流

#### 5. Rich menu 文案

Rich menu 本質上不是單純圖片，而是：

- Chat bar 文案
- 按鈕區塊文案
- 導流結構

LINE 官方產品頁指出 Rich menu 可放最多 `6` 個 links / actions。

所以 Rich menu 文案要先想：

- 用戶最常點什麼
- 你最希望導去哪裡
- 哪些資訊適合放在第一層

#### 6. Broadcast / Rich media / Quick reply / 標準回覆

LINE 官方產品頁與說明中心確認以下文案內容存在：

- Broadcast messages
- Greeting messages
- Auto-response messages
- Standard replies / Quick reply
- Rich messages
- Rich menus
- 1:1 chat 回覆內容

因此若是店家經營實務，最低限度應準備：

- 歡迎訊息文案
- 關鍵字 / 自動回覆文案
- 標準快速回覆文案
- Rich menu 各按鈕名稱
- Rich message CTA 文案
- Broadcast 文案

### B. LINE OA 圖片 / 影片標準尺寸

以下優先整理 LINE Official Account Manager 與 LINE 官方說明頁明確列出的尺寸。

#### 1. Profile 圖片

##### 頭像（Profile picture）

- 格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `3 MB`
- 建議尺寸：`640 x 640 px`
- 顯示上通常以圓形裁切理解較安全

##### Profile Page 封面圖（Profile page cover photo）

- 英文 Help Center 頁面建議尺寸：`1080 x 720 px`
- 泰文規格表頁面另列：`375 x 238 px`
- 實務建議：母檔先做 `1080 x 720 px`，並預留裁切空間，實際上線前再以當前後台預覽確認

##### Cover 圖（PC chat list cover）

- 格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `3 MB`
- 建議尺寸：`1080 x 878 px`
- 僅顯示於 PC chat list
- LINE App `11.9.0` 起不再顯示此 cover

#### 2. Profile Page plug-ins

##### 自由文字外掛（Plug-in Signboard）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 圖片大小：最大 `10 MB`
- 建議尺寸：`688 x 343 px`
- 影片格式：`MP4`、`M4V`、`MOV`、`AVI`、`WMV`
- 影片大小：最大 `200 MB`

##### 內容列表外掛（Plug-in Showcase）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 圖片大小：最大 `10 MB`
- 建議比例：`1:1`

#### 3. Rich message

- 格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 建議尺寸：
  - 自訂模板：`1040 x 520` 到 `1040 x 2080 px`
  - 正方形模板：`1040 x 1040 px`

#### 4. Rich video message

- 格式：`MP4`、`MOV`、`WMV`
- 檔案大小：最大 `200 MB`
- 支援橫式與直式

#### 5. Broadcast 素材

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 圖片大小：最大 `10 MB`
- 影片格式：`MP4`、`M4V`、`MOV`、`AVI`、`WMV`
- 影片大小：最大 `200 MB`
- 語音格式：`WAV`、`MP3`、`M4A`、`AAC`、`OGG`
- 語音大小：最大 `200 MB`

#### 6. Coupon

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 圖片大小：最大 `3 MB`
- 建議尺寸：`640 x 640 px`

#### 7. Reward card

- 背景圖格式：`JPG`、`JPEG`、`PNG`
- 背景圖大小：最大 `10 MB`
- 官方頁面在本次查到的安全描述為：建議使用直式螢幕尺寸，或至少維持 `1:1` 比例

#### 8. Card message

##### 商品卡（Product card）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 建議比例：`1.54 : 1`

##### 地點卡（Location card）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 建議比例：`1.54 : 1`

##### 人物卡（Person card）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 官方頁面有列類型，但我在本次查到的頁面中未看到更明確推薦比例

##### 圖片卡（Image card）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 建議比例：`1.11 : 1`

##### 收尾卡（Ending card）

- 圖片格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 建議比例：`1.54 : 1`

##### 卡片數量

- 內容卡最多 `9` 張
- 收尾卡 `1` 張

#### 9. Rich menu

LINE Official Account Manager 官方建議尺寸：

- 大版：`2500 x 1686`、`1200 x 810`、`800 x 540 px`
- 小版：`2500 x 843`、`1200 x 405`、`800 x 270 px`
- 格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `1 MB`

Messaging API 官方補充規格：

- 格式：`JPEG`、`PNG`
- 寬度：`800` 到 `2500 px`
- 高度：至少 `250 px`
- 長寬比：至少 `1.45`
- 最大檔案大小：`1 MB`

這表示：

- 若在 OA Manager 內建模板做，直接用官方建議尺寸最穩
- 若走 Messaging API，自訂尺寸可更彈性，但仍需落在 API 限制內

#### 10. LINE VOOM

##### 圖片

- 格式：`JPG`、`JPEG`、`PNG`
- 檔案大小：最大 `10 MB`
- 官方頁面註明：高度與長度不受限制

##### 影片

- 格式：`MP4`、`M4V`、`MOV`、`AVI`、`WMV`
- 檔案大小：最大 `500 MB`
- 英文 Help Center 頁面列為：最長 `20` 分鐘
- 泰文規格表另有一列顯示：最長 `1` 分鐘，並另外對 `post video` 列出最長 `20` 分鐘
- 實務建議：若只是一般帳號經營素材，先以 `1` 分鐘內為保守安全值；若走 VOOM / post 專用發佈流程，再依實際後台可選上限操作

##### 預覽圖

- 格式：`JPG`、`JPEG`、`PNG`
- 建議尺寸：`750 x 993 px`

## 四、LINE Developers / Messaging API 相關補充規格

這些不是每個店家在 OA Manager 內都一定會用到，但如果你後續會做更進階訊息編排，這些規格很重要。

### A. Image message

- 原圖 `originalContentUrl`
  - 格式：`JPEG`、`PNG`
  - 最大檔案：`10 MB`
- 預覽圖 `previewImageUrl`
  - 格式：`JPEG`、`PNG`
  - 最大檔案：`1 MB`

官方在這裡沒有規定固定像素尺寸，但要求 HTTPS 與檔案大小限制。

### B. Imagemap message

官方規格：

- 支援格式：`JPEG`、`PNG`
- 每個寬度版本最大檔案：`10 MB`
- 必須提供 5 組寬度版本：
  - `240 px`
  - `300 px`
  - `460 px`
  - `700 px`
  - `1040 px`
- `baseSize.width` 固定為 `1040`
- `baseSize.height` 需依比例對應設定

### C. Flex image / hero image

Messaging API 的 image component 規格：

- 格式：`JPEG`、`PNG`
- 最大圖片尺寸：`1024 x 1024 px`
- 最大檔案：`10 MB`
- 若 `animated = true`，最大 `300 KB`

### D. Flex icon

Messaging API 的 icon component 規格：

- 格式：`JPEG`、`PNG`
- 最大圖片尺寸：`1024 x 1024 px`
- 最大檔案：`1 MB`

### E. Template message 圖片

#### Buttons template

- 格式：`JPEG`、`PNG`
- 最大寬度：`1024 px`
- 最大檔案：`10 MB`

#### Carousel column image

- 格式：`JPEG`、`PNG`
- 比例：`1.51 : 1`
- 最大寬度：`1024 px`
- 最大檔案：`10 MB`

#### Image carousel

- 格式：`JPEG`、`PNG`
- 比例：`1 : 1`
- 最大寬度：`1024 px`
- 最大檔案：`10 MB`

## 五、這次研究後，對你這個課程 / 店家案最實用的素材標準

### A. Google 商家建議實際準備

#### 文案

- 商家名稱：1 組
- 商家描述：1 版（`750` 字元內）
- 服務 / 課程項目：5 到 10 條
- FAQ：8 到 12 題
- Google 貼文文案：
  - Update 3 則
  - Event 1 到 2 則
  - Offer 視是否真有優惠再決定

#### 圖片 / 影片

- Logo：1 張
- Cover 主圖：1 張
- 外觀：3 張
- 教室 / 場地內部：3 張
- 教學中：3 到 6 張
- 實作 / 施工：3 到 6 張
- 講師 / 團隊：3 張
- 30 秒內短片：1 到 3 支

### B. LINE OA 建議實際準備

#### 文案

- 帳號名稱
- Status（20 字內）
- Profile 一句話介紹
- 按鈕文案
- 歡迎訊息 1 組
- 自動回覆 1 組
- Rich menu 按鈕名稱 3 到 6 個
- Rich message 3 張的文案
- Broadcast 2 到 3 則文案
- Coupon 標題 / 說明
- Reward card 說明文案

#### 圖片

- 頭像：`640 x 640`
- Profile Page 封面：先以 `1080 x 720` 母檔製作
- Cover：`1080 x 878`
- Signboard：`688 x 343`
- Coupon：`640 x 640`
- Reward card 背景：以直式螢幕尺寸或至少 `1:1` 比例準備
- Rich menu：
  - 大版 `2500 x 1686`
  - 小版 `2500 x 843`
- Rich message：
  - 正方形 `1040 x 1040`
  - 或長圖寬 `1040`
- Card message：依卡片比例輸出

### Google 商家

#### 必填 / 必核對

- 店名
- 類別
- 地址 / 服務區域
- 電話
- 網站
- 時間

- `JPG/PNG`

## 七、注意事項

### 1. Google 不要把描述寫成廣告

Google 商家描述不是廣告標題區。
不要塞：

- 招生中
- 限時優惠
- 現在只要多少錢
- 電話、網址

### 2. Google 圖片不要過度 AI 化

Google 官方已明寫圖片應代表現實，不應過度濾鏡或 AI 化。
課程 / 教學型商家更應以真實教學照與真實場地照為主。

### 3. LINE 頭像與封面要分工

- 頭像：做辨識
- 封面：做品牌印象
- Rich menu：做導流
- Rich message：做活動 / 課程 / CTA

不要試圖用一張圖解決所有問題。

### 4. Rich menu 不要先做圖再想按鈕

正確順序應該是：

1. 先決定按鈕架構
2. 再決定每個區塊放什麼文案
3. 最後才做對應尺寸的圖
